Build your first website in 20 minutes

June 11, 2017/Kunal Shah/Design, Development
by Kunal Shah

You know have to know HTML or CSS to build a website. In fact, you don't even need to know what they stand for (HyperText Markup Language and Cascading Style Sheets in case you were during to know).

HTML and CSS are markup languages used to write content on the web. they were created to be simple and readable, so that you can focus on design and content rather than syntax and semicolons. HTML is often called the skeleton of a website because it dictates the structure, whereas CSS is the called the skin because it controls the aesthetic elements. Javascript, which I won't be covering in this post, is the muscle of web because it makes things happen behind the scenes.

You can find thorough and intuitive tutorials for each on Codeacademy.com and Udemy.com.

To started however, you don't need to write a single line of code. There are loads of free and open source templates readily available on the web for you to get started with. These are not only the best way to launch a website without having to spend time learning HTML, but they are also a great way to learn about good coding practices and code structure. .

CSS files are often separated from the HTML and linked in the header with 
<link rel="stylesheet" href="css/style.css">

For simplicity, however, the templates above include all the CSS in <style> tags at the top so you don't have to switch between files to see which style elements are modifying which structural elements.

Here's how you get started:
  1. Open a template you like.
  2. Download the source code.
  3. Open the code in a text editor (I recommend Atom or Sublime Text)
  4. Find (Cmd‐F) the text that you can see in the browser and modify it.
  5. Add images into the same folder as your code file.
  6. Change image links.
  7. Boom! your first website.

If you feel like you still don't have a comprehensive understanding of what's going on behind the scenes, don't worry. HTML was designed to do exactly what it looks like it's doing. There is no under the hood magic (unless you're working with Javascript). Now, if there are features you'd like to implement that are not in the template, just google "html " and you'll get more examples on stackoverflow than you thought. A couple examples are "dropdown nav bar" or "background images".

Now, all you need is a domain name.

Luckily for you, domain names usually cost less than $15. The trick is to find something short, snappy, and memorable.


Then, head on to a hosting service, like bluehost.com, and purchase a web hosting subscription. Web hosting is essentially the storage space on the web server that hold your website's files. The domain name you purchase can be directed to point and open the files you store on your web hosting service.


You've done it!

Now, you can iteratively build and update your website by including elements from the web. There are also numerous web design blogs and resources to follow. Codepen is the go-to community for sharing and collaborating on code snippets. Also, Forbes penned a comprehensive list of practical blogs to sharpen your coding skills.
